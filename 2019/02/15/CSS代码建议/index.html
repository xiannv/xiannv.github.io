CSS代码建议

发表于 2019-02-15

1，margin collapse（margin崩塌）

在CSS中，两个或多个毗邻的普通流中的盒子（可能是父子元素，也可能是兄弟元素）在垂直方向上的外边距会发生叠加，这种形成的外边距称之为外边距叠加。

如何避免margin collapse？

  • 采用同一方向的margin值。比如都margin-bottom
  • 触发 block formatting context
    1. float不为none
    2. overflow不为visible
    3. display设为‘table-cell’, ‘table-caption’, 或‘inline-block’
    4. position既不是static也不是relative
    5. zoom:1, IE的hasLayout特性会建立一个新的block formatting context

2，使用flexbox进行布局

传统布局中我们习惯使用floats或者inline-blocks，不过它们更适合于格式化文档，而不是整个网站。而flexbox是专门用于进行布局的工具。

具体用法？

.container{
     display:flex;
     display:-webkit-flex;
}

3，使用css reset

虽然这些年随着浏览器的迅速发展与规范的统一，浏览器特性碎片化的情况有所改善，但是不同浏览器之间仍旧存在很多的行为差异。而解决这种问题的最好方法是使用某个css Reset来为所有的元素设置同意的样式，保证在一个相对统一干净的样式表基础上开始工作。

4，border-box和content-box

默认值为content-box，即当设置某个元素的h/w属性时，仅仅会作用于其内容尺寸，而所有的内边距与边都是在其之上累加120 = (100+2*10)

border-box内边距与边是包含在w/h之中，更加方便进行样式布局，在父元素设置高宽限制而不担心子元素的内边距或者打破这种限制。

5，尽可能使用低优先级的选择器

并不是所有的css选择器的优先级都一样，很多初学者在使用css选择器的时候都是考虑需从新的特性去复写全部的继承特性，不过一旦某个元素多状态就麻烦了：

a#blue-btn{
     background-color:blue;
}
a.active{
     background-color:red
}

id选择器优先级高于class选择器，因此上面代码无效。ID(#id)>Class(.class)>Type(header)

6，em,rem和pixel

适用于不同的场景，不同的开发与项目都有其特定的设置，因此并没有通用的规则来决定应该使用哪个单位：

em 其基本单位即为当前元素的font-size值，经常适用于media-queries中，em是特别适用于响应式开发中。。

rem 其是相对于html属性的单位，可以保证文本段落真正的响应式尺寸特性。？？

px 并没有任何的动态扩展性，它们往往用于描述绝对单位，并且可以在设置值与最终的显示效果之间保留一定的一致性。
